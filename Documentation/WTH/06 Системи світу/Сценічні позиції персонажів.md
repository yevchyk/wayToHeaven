# Сценічні позиції персонажів

Система позицій персонажів у VN-сцені тепер authorиться даними, а не вшитими UI-presets.

## Runtime контракт

Канонічне поле для authoring:
- `stage.characters[].placement`

Поля `placement`:
- `x`: позиція персонажа по ширині сцени у відсотках від `0` до `100`
- `y`: опційний підйом від нижньої межі сцени у відсотках
- `scale`: базовий масштаб портрета
- `zIndex`: ручний шар, якщо персонаж має бути попереду чи позаду інших
- `opacity`: сила присутності персонажа, коли він має бути напівфоновим

## Поведінка runtime

- якщо `placement` не задано, shell використовує widened auto-layout по майже всій ширині сцени
- коли `focusCharacterId` або активний спікер змінюється, portrait отримує підсвітку і додатковий scale boost
- коли `placement.x`, `placement.y` або `placement.scale` змінюються між вузлами, shell анімує перехід через CSS transition
- UI не вирішує, де стоїть персонаж; це приходить із content
- author може вільно "кидати" персонажів по сцені через `x` та `y`, а не лише переставляти їх між умовними слотами
- author може збільшувати або зменшувати персонажа під конкретний beat через `scale`
- `zIndex` і `opacity` дозволяють робити передній план, задній план, напівфонову присутність і сценічний тиск без окремих UI-хаків

## Мінімальний приклад

```json
{
  "stage": {
    "characters": [
      {
        "speakerId": "mirella",
        "emotion": "cool",
        "placement": {
          "x": 18,
          "y": 2,
          "scale": 1.14,
          "zIndex": 5
        }
      },
      {
        "speakerId": "father",
        "emotion": "cold",
        "placement": {
          "x": 82,
          "y": 4,
          "scale": 0.9,
          "zIndex": 2,
          "opacity": 0.78
        }
      }
    ],
    "focusCharacterId": "mirella"
  }
}
```

## Флоу розробки

1. Спершу authorиться сценарний `scene-generation` JSON/TS.
2. Потім у цьому ж runtime-пакеті підганяються `placement`, `focusCharacterId` і scene-level transitions.
3. Після прийняття рішення оновлюється `Documentation`, щоб stage effects, authored coordinates і домовленості по сцені були зафіксовані текстом.

## Обмеження

- `placement.x` поза межами `0..100` невалідний
- `placement.scale` має бути додатнім
- `zIndex` має бути цілим числом
- legacy `left / center / right / extra` не є authoring-контрактом для `sceneGeneration`

## Наслідок для authoring

- якщо в beat персонаж має "наїхати" ближче до камери, це робиться зміною `placement.scale`, а не окремим preset-режимом
- якщо персонажа треба посунути ближче до краю, центру або підняти вгору, це робиться напряму через координати
- якщо сцена потребує різкого перекомпонування, `sceneChange.stage` повністю перевизначає сценічне розкладання персонажів
