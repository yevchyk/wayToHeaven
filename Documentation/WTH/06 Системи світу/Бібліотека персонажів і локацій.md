# Бібліотека персонажів і локацій

## Суть

Це окремий codex-шар для читання персонажів і локацій без втручання в runtime-екрани сцени, міста чи бою. Він потрібен як довідник, а не як місце виконання ігрової логіки.

## Runtime контракт

- `library` це окрема modal-оболонка.
- бібліотека працює поверх поточних контентних реєстрів, а не тримає дубльовану базу даних усередині UI.
- вкладка `Персонажі` читає narrative character registry.
- вкладка `Локації` збирає записи з world locations, city scenes, travel boards і scene-level narrative entries з `sceneRegistry`.
- з початку гри бібліотека не показує весь каталог; записи відкриваються тільки після реальної зустрічі в runtime.
- `SeenContentStore` є persisted source of truth для відкритих codex-записів персонажів і локацій.
- персонаж відкривається, коли він реально входить у beat як `speakerId` або видимий stage-учасник.
- scene-level entry відкривається при вході у відповідний authored scene flow.
- city entry відкривається при вході в hub-flow.
- travel entry відкривається при вході в route-flow.
- world entry відкривається при завантаженні відповідної world location.
- модалка підтримує відкриття на конкретному tab і конкретному entry через payload, щоб workbench і runtime могли deep-link-нути прямо в потрібний запис.
- для Storybook/workbench дозволений явний `showAll` bypass, щоб authoring-режим міг бачити повний каталог без runtime unlock-ів.
- якщо опис або ілюстрація ще не заповнені, UI показує контрольований fallback, а не порожній блок.
- у dialogue HUD має бути прямий вхід у бібліотеку без виходу з поточної сцени.

## Візуальний контракт

- на desktop бібліотека ділиться на selector-column, visual identity block і окремий description panel.
- selector має підтримувати явну підсвітку active entry і пошук по назві, subtitle, description та тегах.
- у звичайному runtime selector показує тільки відкриті записи.
- велика аватарка персонажа або framed location-art лишається окремим візуальним блоком.
- під зображенням лежить identity block з тегами.
- description panel лишається окремим scroll-region.
- у Storybook має існувати окремий workbench для цієї бібліотеки, щоб персонажів і локації можна було клікати й візуально перевіряти без повного runtime-проходу.

## Межі

- бібліотека не виконує ефекти і не змінює сюжетний стан.
- UI не вигадує канонічні дані; він читає те, що вже є в content/docs, і лише добудовує технічний fallback.
- довгі лорні описи мають жити в документації та контенті, а не прямо в React-компонентах.
